Primeiro descrevi o meu novo leitor digital Lev. Em seguida converti os Manuais de Referência de Lua em ePub. Mas agora quero dizer que estes não foram os primeiros documentos que fiz a conversão. Primeiro foi o manual do Rsync. O interessante é que se trata de uma das páginas de manual do Unix (man pages). Portanto, aplicando o mesmo procedimento você pode ter qualquer página de manual no formato ePub.

Experimentei diversas formas, até encontrar uma que me agradasse. Finalmente, o procedimento ficou da seguinte forma. Primeiro converto o manual desejado para o formato HTML utilizando o man2html. Apesar de existirem outros softwares para a tarefa, este foi o que mais me agradou. Na realidade, não me agradou muito. As listas de ítens, por exemplo, ao invés de utilizarem o elemento <ul> é utilizado <dl> usando a letra “o” como marcador. Bastante desagradável, mas melhor que as outras alterativas que experimentei.

Depois uso o Pandoc para converter o HTML em ePub. Assim como descrevi no outro tópico sobre o manual de Lua. E por fim uso o Sigil para editar os detalhes. E aproveitando o tema, também converti este Tutorial do Rsync que já é em HTML mesmo. Foi depois disso que converti os manuais de Lua.

Por fim resolvi converter as traduções De Quando é Ligado até o Bash e Como Construir um Sistema Linux Mínimo do Código Fonte. Usei o pandoc com o formato HTML. Talvez pudesse ter usado o original SGML. No entanto a conversão ficou tão boa que me dei por satisfeito e concluí o trabalho. Fiquei surpreso em quão boa ficou a conversão a partir destes arquivos. Foi a melhor que já realizei até o momento. Segue os arquivos para download:

  • Página de manual do rsync
  • Página de manual do rsyncd.conf
  • Tutorial do rsync
  • De Quando é Ligado até o Bash
  • Como Construir um Sistema Linux Mínimo do Código Fonte
